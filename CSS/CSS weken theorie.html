Onderdeel 1. Selectors & Visual ruler

CSS selector = eerste deel van een CSS-regel.
Het is een patron van elementen en andere termen die de browser vertellen welke HTML elementen moeten worden geselecteerd om de CSS-eigenschappen binnen de regel toe te passen.

Het element of de elementen die door de selector worden geselecteerd, worden 'The subject of the selector' genoemd. Bijvoorbeeld: 
                        h1 {
                            color: blue;
                            background-color: yellow;
                            }

                        p{
                            color: red;
                                }

Styling die toegepast wordt toegepast in een bepaalde volgorde.

Het kan voorkomen dat je styling toegepast en vervolgens niet het gewenste resultaat ziet.

Cascade en Inheritance zijn belangrijke concepten om te begrijpen hoe styling werkt.

Onderdeel 2. Het box model

Alles in CSS heeft een zogenaamde 'box' eromheen het begrijpen van de boxen is sleutel naar lay-outs met CSS te kunnen maken of om items met andere items uit te lijnen.

Term 'box model' gebruikt wanneer we het hebben over ontwerpt en layout. Elke box in CSS bestaat uit 'margins', 'border', 'padding' en 'content' oftwel de inhoud.

Als de box-sizing: border-box gebruikt = dan werkt alles in de box, dus wordt die niet groter naar buiten toe.

Onderdeel 3.Display & Positioning

CSS position property bepaalt hoe een HTML element in een document wordt gepositioneerd.

Je kan elk HTML element op elke gewenste locatie plaatsen.

De top, right, bottom, left properties bepalen de uiteindelijke locatie van geplaatste elementen.

Postioning values Uitleg CSS postioning
Static
Postioning volgens normal flow document.
Top, right, bottom, left, z-index properties have no effect>>default value.
Relative
Postioning volgens normal flow document, offset relative to itself based on values (top, right, bottom, left).
Offset beinvloedt niet de positie van andere elementen, space gegeven voor een element in page layout blijft hetzelfde als static.
Absolute
Element verwijdert van normal flow document, geen space gemaakt voor het element in pagina layout.
Positie is relatief aan de initial containing block.
Final positie is bepaald door values (top, right, bottom, left).
Fixed
Dit value, fixed, maakt een new stacking context. In geprinte documenten, element is gepositioneerd op dezelfde plek op elke pagina.
Element is verwijderd van normal flow document, geen space gemaakt ervoor in pagina layout.
Positie is relatief aan de intial containing block bepaald door viewport, behalve als als het ancestors (transform, perspective, filter) property set is to something other than none. In dit geval reageerd de ancestor als contaning block.
Eind positie wordt bepaald door values (top, right, bottom, left).
Sticky
Postioning volgens normal flow.
Offset is relatief aan zijn dichtsbijzijnde scrolling ancestor en containing block (nearset block-level ancestor).
Incl. table-related elements, gebaseerd op values (top, right, bottom, left). Offset beinvloed niet de positie van andere elementen.
Deze value maakt een new stacking context.
Sticky element plakt aan de dichtsbijzijnde ancestor en heeft scrolling mechanisme (gemaakt overflow is hidden, scroll, auto, overlay), zelfs als dichtsbijzijnde ancester geen dichtsbijzijnde scrolling ancestor is.
Effectively inhibits any sticky behaviour.
            De default position in je html is static. Het element wordt gepositioneerd volgens de normale stroom van het document. De eigenschappen boven, rechts, onder, links en z-index hebben geen effect. Dit is de standaardwaarde.
                
            Position properties:
            Relative: Het element is relatief gepositioneerd ten opzichte van zijn normale positie, dus "left: 20px" voegt 20 pixels toe aan de LINKER-positie van het element
                
            Absolute: Het element is gepositioneerd ten opzichte van het eerste gepositioneerde (niet statische) bovenliggende element
                
            Fixed: Het element is gepositioneerd ten opzichte van het browservenster
                
            Sticky: Het element wordt gepositioneerd op basis van de scrollpositie van de gebruiker.

Onderdeel 4. Pseudo elements

Met CSS kan je stijlen definieren op basis van elementen. Maar soms is het niet genoeg om het gewenste resultaat te krijgen. Als je alleen over element- of attribuut- selectors hebt, bijvoorbeeld opmaak eerste letter/regel van inhoud elementen.

Om dit mogelijk te maken zijn Pseudo-elementen geintorduceerd.

Je kan ze zien als een denkbeeldig element die niet in je document voorkomt, maar wel kan 'stijlen'.

Je tal van interessante ontwerpmogelijkheden toepassen zonder de senmatiek van de opmaak negatief te beinvloeden.

De meest gebruikte pseudo-elementen zijn: "::before en ::after".

Pseudo elementen kan je position: absolute, relative t.o.v. parent element, kan je ze zien als twee extra lagen waar je me kunt spelen.

Tal van toepassingen mogelijk, zoals meervoudige borders, achtergronden, allerlei vormen.

Uitleg meerdere achtergrondne en borders (niet beveiligd).

Pseudo-elements
Keyword added to a selector that lets you style a specific part of the selected element(S).
Bijvoorbeel: p::first-line{____} = verandert eerste regel font van een paragraaf
Syntax pseudo-elements

Pseudo classes

Pseudo Classes
A keyword added toa selector that specifies a special state of the selected element(S).
Can apply a style to an element not only in relation to the content of the document tree, but also in relation to external factors like history of the navigator (:visitied), status of content checked (:checked) or position of the mouse (:hover).

Linguistic pseudo-class

:dir
Selects a element based on its directionality as determined by document language.
: lang
Select an element based on its content language.

Location pseudo-class

:any-link
Matches an element if the element would match either :link or :visited.
:link
Matches links that have not yet been visited.
:local-link
Matches links whose absolute URL is the same as target URL, for example anchor links to the same page.
:target
Matches the element which is the target of the document URL.
:target-within
Matches elements which are the target of the document URL, but also elmeents which have a descendant which is the target of the document URL.
:scope
Represents elements that are a reference point for selectors to match against.

User-Action pseudo-class

:hover
Matches when a user designates an item with a pointing device, for example holding the mouse pointer over it.
:active
Matches when an item is being activated by the user, for example clicked on.
:focus
Matches when an element has focus.
:focus-visible
Matches when an element has focus and the user agent identifies that the element should be visibly focused.
:focus-within
Matches an element to which :focus applies, plus any element that has a descendant to which :focus applies.

Time-dimensional pseudo-class

:current
Represents the element or ancestor of the element that is being displayed.
:past
Represents an element that occurs entirely before the :current element.
:future
Represents an element that occurs entirely after the :current element.

Resource state pseudo-class

:playing
Represents a media element that is capable of playing when that element is playing.
:paused
Represents a media element that is capable of playing when that element is paused.

Input pseudo-class

:enabled
Represents a user interface element that is in an enabled state.
:disabled
Represents a user interface element that is in a disabled state.
:read-only
Represents any element that cannot be changed by the user.
:read-write
Represents any element that is user-editable.
:placeholder-shown
Matches an input element that is displaying placeholder text, for example from the HTML5 placeholder attribute.
:default
Matches one or more UI elements that are the default among a set of elements.
:checked
Matches when elements such as checkboxes and radiobuttons are toggled on.
:indeterminate
Matches when UI elements are in an undeterminate state.
:blank
Matches a user-input element which is empty, containing an empty string or other null input.
:valid
Matches an element with valid contents. For example an input element with type 'email' which contains a validly formed email address.
:invalid
Matches an element with invalid contents. For example an input element with type 'email' with a name entered.
in-range
Applies to elements with range limitations, for example a slider control, when the selected value is in the allowed range.
:out-of-range
Applies to elements with range limitations, for example a slider control, when the selected value is outside the allowed range.
:required
Matches when a form element is required.
:optional
Matches when a form element is optional.
:user-invalid
Represents an element with incorrect input, but only when the user has interacted with it.

Tree-structural pseudo-class

:root
Represents an element that is the root of the document. In HTML this is usually the element.
:empty
Represents an element with no children other than white-space characters.
nth-child
Uses An+B notation to select elements from a list of sibling elements.
:nth-last-child
Uses An+B notation to select elements from a list of sibling elements, counting backwards from the end of the list.
:first-child
Matches an element that is the first of its siblings.
:last-child
Matches an element that is the last of its siblings.
only-child
Matches an element that has no siblings. For example a list item with no other list items in that list.
:nth-of-type
Uses An+B notation to select elements from a list of sibling elements that match a certain type from a list of sibling elements.
:nth-last-of-type
Uses An+B notation to select elements from a list of sibling elements that match a certain type from a list of sibling elements counting backards from the end of the list.
:first-of-type
Matches an element that is the first of its siblings, and also matches a certain type selector.
:last-of-type
Matches an element that is the last of its siblings, and also matches a certain type selector.
:only-of-type
Matches an element that has no siblings of the chosen type selector.

Onderdeel 5. Flexbox

Uitleg CSS flex
Uitleg complete gids Flexbox
Flexbox
Is CSS model voor het positioneren van HTML elementen.
Ideale techniek voor ontwikkeling van een responsive layout.
Gebruik van Float en Absolute positoning is met deze techniek verleden tijd.
Grote voordeel is dat het een volwaardig CSS model is die je volledige controle geeft over de zaken belangrijk zijn bij een responsive layout van je webpagina.
Bijvoorbeeld, uitlijning/sortering van HTML elementen.
Flex container
Idee erachter is om de container de mogelijkheid te geven de breedte- /hoogte (en volgorde) van de items te wijzigen om de beschikbare ruimte zo goed mogelijk te vullen (voor alle schermformaten en displays).
Belangrijkste is dat de lay-out van de flexbox niet richtingsafhankelijk is, in tegenstelling toegepas tot reguliere lay-outs (block dat verticaal is gebaseerd en inline block dat horizontaal is gebaseerd.)

Onderdeel 6. Transitions

Uitleg CSS transitions
Uitleg Animatable CSS properties
CSS transitions
Biedt een manier om animatiesnelheid te regelen bij het wijzigen van CSS-properties.
Zorg ervoor dat bepaalde CSS/styling niet onmiddelijk worden toegepast.
Toegpast: zorg voor wijzigen plaats met tijdsintervallen die een versnellingscurve volgen, die allemaal kunnen worden aangepast naar wens.
Implicit transitions
Animaties die een transistie tussen twee toestandne met zich meebrengen.
De staat tussen start- en eindstaat impliciet wordt gedifineerd door de browser.
Notes
hoelang transitie zal duren (duration instellen) en hoe transitie zal verlopen (timing function te definieren, bijvoorbeeld ease-in-out).
Je kan zelf beslissen welke properties jij wilt animeren (expliciet vermelden), wanneer de animatie zal starten (delay instellen).
